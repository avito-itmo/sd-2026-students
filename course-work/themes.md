# Темы проектных работ по курсу «Системное проектирование ПО»

## 1. Сервис уведомлений

**Контекст:**  
Сервис уведомлений отвечает за доставку сообщений пользователям по различным каналам: email, SMS, push-уведомления,
webhooks. Он получает события из других систем и решает, какие уведомления, в каком формате и по каким каналам
необходимо отправить.

### Уровень сложности
Базовый — простой домен, высокая интеграционность, удобен для первого архитектурного проекта.

### Архитектурные стили
 - Event-driven architecture
 - Асинхронное взаимодействие
 - Message broker как инфраструктурный элемент

### Декомпозиция
 - Inbound Event Listener
 - Notification Orchestrator
 - Channel Providers (Email, SMS, Push)

### Нефункциональные требования
 - Надёжность доставки
 - Масштабируемость
 - Idempotency

### DDD
 - Bounded Context: Notifications
 - Value Object: Message, Channel
 - Domain Events

### BDD
 - Сценарии доставки уведомлений
 - Ошибки каналов
 - Повторные попытки

### Эволюция архитектуры
 - Добавление новых каналов
 - Изменение правил маршрутизации
 - Переход к multi-tenant

### Архитектурная документация
 - ADR по выбору брокера
 - C4: контейнеры и компоненты
